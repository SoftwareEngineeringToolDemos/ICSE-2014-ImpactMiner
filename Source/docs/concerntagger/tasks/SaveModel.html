Save the concern model

Actions:

  1. Select the SaveAs button.
  2. In the file dialog, choose a file in which to save the concern model or type the name of a new file.

Notes:

  • If the model contains unsaved data, you can select the Save button to save the concern model in its current file.
  • If the current concern model's file has been moved or renamed, the Save As... dialog will open to specify a new file and location for saving the concern model..
  • If the specified file already exists you are asked to confirm the override.
  • Saving errors can happen if elements of the concern model are of unsupported types. For example, it is not possible to save local or abstract methods. In this case the elements that can't be saved will be skipped and saving will continue.
  • Unsaved changes to the concern model will be lost when Eclipse is closed: remember to save frequently.

Related Preferences:

  • Always save with a ".cm" file extension
  • Automatically save the concern model when the workbench is closed
