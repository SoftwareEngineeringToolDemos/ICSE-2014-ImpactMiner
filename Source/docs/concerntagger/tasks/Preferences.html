Preferences

Accessing the ConcernMapper preference page


  1. Access the eclipse preferences page through Window/Preferences....
  2. Select the ConcernMapper preference page from the left pane.

Filter:

When enabled, elements below the filtering threshold will not be displayed in the ConcernMapper view, altough they will still be part of the concern model.

Enable/disable filter

  1. Select/deselect the checkbox to enable/diable the filter.

Set the threshold degree

  1. Type an integer between 1 and 99 in the Threshold degree textbox.

Related Tasks:

  • Filter Elements below a certain threshold
  • Modify and element's degree

Saving:

Always save concern models with a ".cm" file extension

  1. Check Always save concern model with .cm file extension.

Notes:

  • When this preference is enabled, if the file name entered by the user is *.cm, the file is saved as *.cm .
  • When this preference is enabled, if the file name entered by the user is *.*, the file is saved as *.cm .
  • When this preference is enabled, if the file name entered by the user does not have a file extension specified, the file is saved as *.cm .
  • When this preference is disabled, the file is always saved as entered by the user.
  • Files that do not have a .cm file extension cannot be viewed in the ConcernMapper view.

Automatically save the concern model when the workbench is closed

  1. Check Automatically save the concern model when the workbench is closed.

Notes:

  • When this preference is enabled, if a concern model model that has been previously saved contains unsaved changed, the model will be saved to the file it had previously been saved in.
  • When this preference is enabled, if a previously unsaved concern model model is active, the Save As dialog will appear before the workbench closes.
  • If a concern model is empty, the last saved version of that concern model will not be overwritten.

Related Tasks:

  • Save the concern model

Automatically load the last concern model when the workbench is opened

  1. Check Automatically load the last concern model when the workbench is opened.

Notes:

  • When this preference is enabled, the concern model that was previously opened in the ConcernMapper view will be automatically loaded when the workbench is opened.

Related Tasks:

  • Load a concern model

Decorations:

Enable suffixes

  1. Select/deselect the checkbox to enable adding the names of concerns as suffixes.

Enable bold font

  1. Select/deselect the checkbox to enable displaying concern model elements in bold font.

Decorate parents

  1. Select the radio button to choose to which level parents of concern elements should be decorated.

Note:

  • Decorations are always enabled by default when ConcernMapper view is opened. To disable decorations altogether, access the Eclipse preferences page under General/Appearance/LabelDecorations and disable ConcernMapper Decorator.
